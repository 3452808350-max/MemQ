# 🛡️ Bug 记忆防重复机制

**创建时间**: 2026-03-13  
**问题**: 重复归档造成的浪费（时间 + 存储 + 检索困难）  
**解决方案**: 归档前查重 + 特征指纹 + 智能匹配

---

## 🔄 浪费场景

### 问题流程

```
遇到 Bug
  ↓
检索历史 Bug
  ↓
❌ 检索失败（关键词不匹配）
  ↓
重新分析根因（浪费时间）
  ↓
重新记录 Bug
  ↓
❌ 重复归档（浪费存储）
  ↓
Bug 库膨胀（检索更困难）
  ↓
恶性循环...
```

### 浪费统计

| 浪费类型 | 说明 | 影响 |
|---------|------|------|
| **时间浪费** | 重新分析已解决的问题 | 25 分钟/次 |
| **存储浪费** | 重复数据 | 2x 存储 |

## ✅ 防重复机制

### 1. 归档前查重

**工具**: `check_bug_duplicate.py`

**流程**:
```
新 Bug 报告
  ↓
提取特征（症状、系统、操作）
  ↓
检索现有 Bug
  ↓
计算相似度
  ↓
相似度 ≥ 70%？
  ├─ 是 → ⚠️ 警告：可能重复，查看相似 Bug
  └─ 否 → ✅ 可以归档，生成特征指纹
```

---

### 2. 特征指纹

**原理**: 提取关键特征生成唯一指纹

```python
# 关键特征
features = [
    symptom,      # 症状
    system,       # 系统
    operation     # 操作
]

# 生成指纹
fingerprint = MD5('|'.join(features))[:12]
# 示例：c96b2833719b
```

**用途**:
- 快速查重（比对指纹）
- 识别重复（相同指纹）
- 去重统计

---

### 3. 智能相似度计算

**多维度匹配**:

| 维度 | 权重 | 说明 |
|------|------|------|
| **症状相似度** | 50% | 文本相似度算法 |
| **描述相似度** | 30% | 整体描述匹配 |
| **系统匹配** | 20% | 是否同一系统 |

**阈值**:
- ≥ 70%: 高度相似（可能重复）
- 50-70%: 中度相似（参考）
- < 50%: 低度相似（新问题）

---

### 4. 归档流程优化

#### 旧流程（有问题）

```
记录 Bug → 直接归档 ❌
```

#### 新流程（防重复）

```
记录 Bug
  ↓
运行查重工具
  ↓
发现相似 Bug？
  ├─ 是 → 查看相似 Bug → 确认是否重复
  │        ├─ 是重复 → 更新现有 Bug（不新增）
  │        └─ 不重复 → 继续归档
  └─ 否 → 生成特征指纹 → 归档 ✅
```

  --symptom "召回率 80%" \

## 📊 防重复策略

### 1. 强制查重

**规则**: 归档前必须运行查重

```bash
# 归档脚本中自动调用
python3 check_bug_duplicate.py \
  --symptom "$SYMPTOM" \
  --system "$SYSTEM"

# 确认不重复后再归档
```

---

### 2. 特征指纹索引

**文件**: `.bug_fingerprints.json`

```json
{
  "c96b2833719b": "BUG-20260313-001",
  "a8f3d92e41bc": "BUG-20260314-001",
  ...
}
```

**用途**:
- 快速查重（比对指纹）
- 统计去重
- 识别重复提交

---

### 3. 定期清理

**任务**: 每月检查重复 Bug

```bash
# 查找重复指纹
python3 find_duplicate_bugs.py

# 合并重复 Bug
python3 merge_duplicate_bugs.py
```

---

### 4. 更新而非新增

**场景**: 同一 Bug 再次出现

**旧方式**: 新增 Bug 记录 ❌

**新方式**: 更新现有 Bug ✅

```json
{
  "bug_id": "BUG-20260313-001",
  "occurrences": [
    {
      "date": "2026-03-13",
      "context": "首次出现"
    },
    {
      "date": "2026-03-20",
      "context": "再次出现，已解决"
    }
  ],
  "status": "resolved"
}
```
