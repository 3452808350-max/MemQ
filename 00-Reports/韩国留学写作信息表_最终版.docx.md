### 重点项目：memory-lancedb-pro

**问题发现**：在使用 OpenClaw 时，我发现 AI Agent 经常遗忘之前对话中的重要信息，因为上下文窗口有限。这从根本上限制了 Agent 建立长期理解的能力。

**探索过程**：这个问题驱使我研究 AI Agent 长期记忆系统。我学习了检索增强生成（RAG）、向量数据库相关的论文和技术文档。

**技术发现**：通过实验，我发现纯向量检索虽然在语义匹配方面表现良好，但在需要精确关键词匹配的场景下经常失效——在我的测试中召回率只有 68%。

**系统设计**：为解决这个局限，我设计并实现了 memory-lancedb-pro，一个混合检索记忆系统。系统结合了向量搜索（语义理解）、BM25（精确关键词匹配）、Cross-Encoder Rerank（相关性优化）、时间衰减（优先近期信息）和噪声过滤（排除低质量记忆）。

**实现与验证**：该系统将对话记忆存储在 LanceDB 中，动态检索相关信息以扩展 Agent 的有效上下文。通过使用倒数排名融合（RRF）融合向量和 BM25 结果，并应用 cross-encoder 重排序，我将检索召回率从 68% 提升到 78%——提升了 15%。项目已在 GitHub 开源。

**收获与反思**：这个项目教会了我三件事：第一，真实的 AI 系统不仅仅是调用 API，理解底层算法和工程权衡至关重要；第二，混合方法往往优于单一方法；第三，评估很重要，没有基准测试就无法知道改进是否有意义。

同时，这个经历也暴露了我的知识差距。虽然我能够实现和优化现有方法，但我缺乏深厚的理论基础。为什么 RRF 比简单分数融合更有效？Cross-Encoder 的工作原理是什么？如何设计新颖的架构？这些问题驱使我想要系统地学习 AI 理论。
