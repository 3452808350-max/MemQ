## 🏗️ 核心技术

### 1. 分层记忆架构

- **L0 摘要层**: ~100 tokens, 节省 81%
- **L1 概览层**: ~2K tokens, 节省 97%
- **L2 详细层**: 完整内容，按需加载

### 2. 混合检索引擎

**流程**: BM25+ 向量 → RRF 融合 → 质量分加权

- **BM25**: 关键词检索
- **向量**: Ollama Qwen3-Embedding-0.6B
- **RRF**: 倒数秩融合
- **质量分**: 6 维度评分加权

### 3. 6 维度质量评分

| 维度 | 权重 | 说明 |
|------|------|------|
| **类型** | 10% | 代码/技术内容高分 |
| **长度** | 20% | 200-2000 字符最佳 |
| **实体密度** | 10% | URL/邮箱/代码/数字 |
| **破坏词** | 30% | "抱歉"/"无法"/"作为 AI" |
| **模板化** | 25% | "好的，..."/"让我..." |
| **元数据** | 5% | 分类权重 |

### 4. 智能缓存

- **持久化**: JSON 文件
- **TTL**: 7 天
- **LRU**: 最大 1000 条
- **预热**: 启动时预计算

## 📈 关键发现

### 1. 混合检索效果显著

- BM25+ 向量 → RRF 融合
- Recall@1 从 60% 提升到 90%
- 语义理解大幅改善

### 2. Rerank 可省略

- 无 Rerank 也能达到 90% Recall@1
- 速度更快（200ms vs 500ms+）
- 推荐配置：`use_rerank=False`

### 3. 噪声识别需要多规则

- 单纯调整阈值不够
- 需要特殊规则处理：
  - 纯问候语（"你好"、"早上好"）
  - 填充词（"嗯"、"啊"）
  - 省略号（"..."）
  - 重复词（"嗯嗯嗯"）

### 4. 质量阈值 0.6-0.7 最佳

- <0.4: 太宽松，噪声通过
- 0.6-0.7: 最佳平衡点
- >0.8: 可能过严，误杀高质量

## 🔧 使用方式

### Python 调用

```python
from memq_pro import MemQPro

memq = MemQPro()

# 保存记忆
memq.add_memory("pref", "喜欢简洁代码", "preferences")

# 检索（混合检索 + 质量分）
results = memq.search(
    "代码风格",
    top_k=5,
    use_hybrid=True,      # 混合检索
    use_rerank=False,     # 无 Rerank（快）
    use_quality=True,     # 质量分加权
    min_quality=0.6       # 最低质量阈值
)
```

### HTTP API

```bash
# 启动服务
python3 memq_bridge.py

# 搜索
curl -X POST http://localhost:5555/search \
  -H "Content-Type: application/json" \
  -d '{"query":"代码风格","top_k":5}'
```

```

/memq_stats
```
